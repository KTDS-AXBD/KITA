### 2.5 제출 처리 — Google Form 백엔드 POST (✅ 결정됨)

09번 HTML에 (a) 기존 클립보드+JSON 옵션은 그대로 유지하고, (b) **"Google Form으로 직접 제출"** 버튼을 추가해 자동 수집.

#### 구현 절차

**Step 1. Google Form 생성**
- Google Form 새 폼 만들기 → 09번 HTML의 모든 질문을 1:1로 추가
  - 응답자 정보 3개 (이름·소속·역할 / 응답일자 / 대표 회신자)
  - A. 우선순위 — 시나리오 7개 × 1~3순위 (드롭다운 7개) + 선정 기준 3개 + 추가 시나리오 텍스트
  - B. 데이터 의향 — 항목 10개 × 즉시/일부/추후/불가 (라디오 그리드 가능)
  - C. OK 기준 — L1~L5 체크박스 + 청중 + 시연 형식
  - D. 일정 — 일자 체크박스 + 시간대 + B&F 방식 + PM 정보
  - E. 자유 의견 × 2
- 폼 미리보기 URL: `https://docs.google.com/forms/d/e/{FORM_ID}/viewform`
- 제출 엔드포인트: `https://docs.google.com/forms/d/e/{FORM_ID}/formResponse`

**Step 2. entry.* ID 추출**
폼 미리보기에서 개발자 도구 → Network 탭 → 임의 입력 후 제출 → `formResponse` 요청의 form-data에서 각 필드별 `entry.{ID}` 추출. 폼별 고유.

또는 미리보기 페이지의 HTML 소스에서 `data-params` 안에 `entry.{ID}` 매핑 검색.

권장 도구: 브라우저 확장 "Get Form Field IDs" 또는 수동 추출. 매핑 표 작성:

```js
// src/lib/googleFormMap.ts (또는 09번 HTML 안 <script>)
const GOOGLE_FORM = {
  url: 'https://docs.google.com/forms/d/e/__FORM_ID__/formResponse',
  fields: {
    respondent:  'entry.111111111',
    resp_date:   'entry.222222222',   // YYYY-MM-DD
    rep:         'entry.333333333',
    prio_s1:     'entry.444444444',
    // ... 전체 매핑 (TODO: 폼 생성 후 채움)
  }
};
```

**Step 3. POST 전송**

CORS 우회를 위해 `mode: 'no-cors'` 사용 (응답을 못 받지만 전송은 됨):

```js
async function submitToGoogleForm(data) {
  const fd = new FormData();
  for (const [key, entryId] of Object.entries(GOOGLE_FORM.fields)) {
    const v = data[key];
    if (v === undefined || v === '') continue;
    if (Array.isArray(v)) v.forEach(item => fd.append(entryId, item));
    else fd.append(entryId, v);
  }
  await fetch(GOOGLE_FORM.url, { method: 'POST', mode: 'no-cors', body: fd });
  // no-cors는 응답을 못 읽으니 성공 가정. 실제 전송 여부는 Google Form 응답 시트에서 확인
}
```

**Step 4. 응답 시트 자동 수집**
- Google Form → 응답 탭 → 스프레드시트 연결
- KT DS 내부 공유 폴더에 저장
- 필요 시 Apps Script 트리거로 Slack 알림 추가:
  ```js
  function onFormSubmit(e) {
    const values = e.values;
    const slackUrl = PropertiesService.getScriptProperties().getProperty('SLACK_WEBHOOK');
    UrlFetchApp.fetch(slackUrl, {
      method: 'post',
      contentType: 'application/json',
      payload: JSON.stringify({ text: `새 회신 도착: ${values[1]}` })
    });
  }
  ```

#### 09번 HTML에 추가할 변경

- 헤더 안내문에 "제출 시 KT DS 응답 시트로 자동 전송됩니다" 한 줄 추가
- 제출 버튼 옆에 "Google Form으로 직접 제출" 보조 버튼 추가 (현재 미리보기 + 클립보드/JSON 옵션은 유지)
- `GOOGLE_FORM.fields` 매핑이 채워지기 전까지는 보조 버튼 클릭 시 "폼 ID 미설정" 안내. Sprint 0 마지막 단계에서 폼 생성 후 ID만 채우면 활성화

#### 보안·운영 메모

- Google Form은 응답자 인증 없이 받음 → 봇·중복 응답 가능. 회수 채널을 시연 직후 1:1 발송으로 한정해 위험 낮춤
- 응답 시트는 KT DS 도메인 계정 소유로 생성, 기진회 측에는 결과 요약만 공유
- 폼 자체에 "본 폼은 PoC 준비용이며 응답 내용은 외부로 공유되지 않습니다" 안내 명시

### 3.6 알려진 제약

- iframe 안 컨텐츠는 사이트 디자인 토큰 변경 시 별도 갱신 필요 (axis 토큰을 09번에 하드코딩했으므로 토큰 시스템이 크게 바뀌면 09번도 손봐야 함)
- iframe 안 폼은 사이트 다크모드 토글 등에 자동 반응 안 함 — 단독 라이트 모드 고정
- 모바일 뷰에서 iframe 높이가 viewport에 종속되므로 `calc(100vh - 120px)` 정도로 잡되, 폼이 더 길면 폼 자체 스크롤
- 본 사업화 단계에서는 C(React 정식 이식)로 옮겨야 axis Dialog·Form·Select 등과 일관성 확보
