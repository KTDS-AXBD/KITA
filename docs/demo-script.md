# KOAMI PoC 시연 스크립트 (F011 · GIVC Ontology Platform v0.32)

> Prototype 리뷰 시연용 **핵심 흐름 outline** (25~35분). 대상: **한국기계산업진흥회(KOAMI) 소부장·가치사슬 담당 + 회원사**. 발표자: 서민원 책임.
> 핵심 목표: 고객이 **이미 보유한 chatGIVC(LLM+RAG)** 대비 **온톨로지 + Knowledge Graph**의 우월성(인과추적·설명가능성·재현성·의사결정 지원)을 **데이터→질문→구조→추론→비교** 여정으로 증명 + **고객이 자발적으로 보유 데이터를 꺼내게** 유도.
> 사전: 라이브 가용 확인(`bash scripts/healthcheck.sh https://koami.minu.best`) + CF Access 로그인 준비(인증 브라우저) + 오프라인 백업 대기(`pnpm serve:offline`).

---

## 흐름 한눈에

```
진입 /platform/data → ① 데이터 현황 (3~4분) → ② CQ 관리 (3~4분) → ③ 온톨로지 정의 (3~4분)
→ ④ 지식그래프 (4~5분) → ⑤ 시나리오 분석 (8~10분, 하이라이트) → ⑥ 비교 검증 (4~5분, 전략 핵심)
→ ⑦ 추진 계획 (2~3분) + 마무리·데이터 유도
```

- **화면**: 사이드바 대시보드 7페이지(`https://koami.minu.best/platform/data`부터). 사이드바로 순서대로 이동.
- **데이터 표기**: 모든 표·그래프에 **실(검정) / 추정(회색) / 유료(연회색)** 배지. "유료 = 본사업 단계 도입 검토 데이터"를 자연스럽게 노출.
- **기본 도메인**: 소부장·공작기계. 호르무즈(석유화학)는 ④⑤에서 **토글**로 "같은 엔진이 전 산업·지정학 충격까지" 시연.
- **서사 한 줄**: **"데이터를 질문(CQ)으로, 질문을 온톨로지·그래프로, 그래프를 인과 추론·의사결정으로 — chatGIVC가 못 하는 걸 합니다. 6월에 실 GIVC로 본구축."**

> 참고: 기존 단일 화면(`/scenario/rnd`·`/scenario/s6`·`/about`·`/survey`)은 라우트 유지(공존). 본 시연은 신규 7페이지 플랫폼 중심.

## ① 데이터 현황 — "재료 점검" (3~4분)

- **화면**: `/platform/data`
- **핵심 메시지**:
  - KPI 4장 — **총 27개 데이터 소스 · 실 19(70%) · 추정 4 · 유료 4**. "분석에 쓰는 데이터를 먼저 투명하게 펼칩니다."
  - 27건 현황표(상태닷·구분 배지·출처·활용 영역·수집 방법·최종 갱신) — "GIVC 기보유 + 공개 API + 유료가 한 표에. 연동 상태까지 관리합니다."
  - 배지 짚기: "**실** = 검증 가능한 실데이터(GIVC DB·공공 API), **추정** = API 존재하나 보강 전, **유료** = 구독/라이선스 — 본사업에서 도입 검토."
- **데이터 유도** 🎯 C1: "여기 **유료/추정**으로 잡힌 항목들 — 귀측이 이미 보유하셨다면 바로 **실**로 승격됩니다. 어떤 게 사내에 있을까요?"

## ② CQ 관리 — "무엇에 답할 것인가" (3~4분)

- **화면**: `/platform/cq`
- **핵심 메시지**:
  - 좌측 CQ 목록 + 필터(전체/Tier1/Tier2/검증완료/미검증) — "**Competency Question** = 온톨로지가 답할 수 있어야 하는 질문. 시스템을 질문으로 정의합니다."
  - **Tier1(시연 대상)**: CQ-001 호르무즈 영향 품목, **CQ-002 소부장 자립화 R&D 적합 기업**. **Tier2(고객 확인 후)**: 수입 다변화·EWS 전파·국산화율 등 5건.
  - CQ 클릭 → 우측 상세(질문·배경 Pain Point·관련 엔티티·**Cypher 쿼리**·검증 결과). "질문마다 어떤 데이터·관계로 답하는지가 명시됩니다 — 블랙박스가 아닙니다."
  - (선택) 신규 CQ 등록 모달 열기 — "현장에서 나온 질문을 바로 등록할 수 있습니다."
- **데이터 유도** 🎯 C2: "**Tier2는 고객 확인 후 추가**입니다 — 귀측이 가장 답받고 싶은 질문은 무엇인가요? 그게 다음 CQ가 됩니다."

## ③ 온톨로지 정의 — "구조" (3~4분)

- **화면**: `/platform/ontology`
- **핵심 메시지**:
  - KPI — **엔티티 13종 · 관계 8종 · 인스턴스 161 · 관계 331**.
  - 엔티티 카드 13종(Event·Region·Country·RawMaterial·IntermediateGoods·Product·Industry·Company·EWS·Policy·R&D·Supply·Risk) — 색상·속성·툴팁. "도메인을 명사(엔티티)와 동사(관계)로 모델링합니다."
  - 관계 테이블 8행(DISRUPTS·INPUT_TO·REFINES_TO·PRODUCES 등, 클릭 → 상세 모달) + **제약 다크블록 3종**(유일성·NOT NULL·값범위). "제약으로 데이터 품질을 강제 — 검색이 아니라 추론의 기반입니다."
- **메시지 포인트**: "이 구조가 chatGIVC와의 결정적 차이입니다. 문서를 검색하는 게 아니라, **관계를 따라 추론**합니다."

## ④ 지식그래프 — "인과를 본다" (4~5분)

- **화면**: `/platform/graph`
- **조작**:
  1. **소부장 그래프**(기본, 37노드) — cytoscape 인과 그래프. 노드 클릭 → 우측 **상세 패널**(속성·출처·연결). "공작기계 ← 부품(베어링·감속기) ← 소재(특수강) 전후방을 그래프로."
  2. **노드 필터 / 영향경로 토글** — "관심 경로만 강조. 50노드 이하로 한눈에."
  3. ⭐ **도메인 토글 (소부장 ↔ 호르무즈)** — "같은 엔진이 **석유화학·지정학 충격**(44노드)으로. 품목 무관 — 구조가 같으면 그대로 재렌더됩니다."
- **데이터 유도** 🎯 C3: "이 그래프 노드·관계는 공개데이터+가상 재현입니다. **귀측 GIVC 실데이터**(전후방·기업)를 연결하면 이 화면이 그대로 실값으로."

## ⑤ 시나리오 분석 — 의사결정 지원 (8~10분, ★ 하이라이트)

- **화면**: `/platform/scenario` (기본 **CQ-002 소부장**)
- **조작 순서**:
  1. **CQ 선택** = CQ-002 소부장 자립화 R&D — "산기평 공고가 뜨면, 적합 기업 Top5 + 선정 근거 + 반대 추천을 GIVC 데이터로."
  2. **NL 질의 → Cypher 변환 보기** 토글 — "자연어 질문이 그래프 쿼리로 변환됩니다. (시연용 결정적 스크립트 — 같은 질문 = 같은 결과, **재현성**)"
  3. **분석 실행** → **5단계 애니메이션 추론** — "충격/공고 → 데이터 수집 → 관계 추적 → 후보 산출 → 근거 정리. 추론 과정을 **보여줍니다**."
  4. 결과 **A 영향/추천 경로**(cytoscape mini) · **B Top5 표**(대한정밀감속기·서원베어링테크·미래기계소재 등, 점수·근거·반대추천) · **C 설명가능성**(인과 경로·취약성 지표·EWS 경보·**재현성 ID**) · **D 대응 옵션 비교**(실행가능성·기간·효과·리스크) · **E 의사결정 리포트**(Executive Summary + 단계별 전파 프레임워크).
  5. ⭐ **CQ 토글 → CQ-001 호르무즈** — "같은 분석 틀이 지정학 충격에도. 호르무즈 봉쇄 시 석유화학 품목 Top5(PE·PP…)·인과 경로·대응 옵션이 동일 구조로. **하나의 엔진, 모든 시나리오**."
- **데이터 유도** 🎯 C4: "지금은 큐레이션 데이터로 추론을 **재현**합니다. 실 GIVC에 직결되면 이 리포트가 실시간 실값으로 나옵니다 — 어떤 시나리오부터 보고 싶으신가요?"

## ⑥ 비교 검증 — chatGIVC vs 온톨로지+KG (4~5분, ★ 전략 핵심)

- **화면**: `/platform/compare`
- **핵심 메시지** (여기가 설득의 정점):
  - 좌 카드 **chatGIVC(LLM+RAG)**: 같은 질문에 "일반론 답변 + 문서 출처"만 — ✗ 구체 품목 없음 · ✗ 인과 경로 없음 · ✗ 정량 근거 없음 · ✗ 대응 옵션 불가.
  - 우 카드 **온톨로지+KG**: ✓ 품목+순위+점수 · ✓ 다단계 인과 경로 자동 추적 · ✓ 투입계수/의존도 정량 · ✓ 대응 옵션+근거+리스크 · ✓ 설명가능성 · ✓ 재현성.
  - **6축 비교표**: 인과관계 추적 · 설명가능성 · 재현성 · 시나리오 예측 · 대응 옵션 · 데이터 신뢰.
  - 멘트: "**chatGIVC는 이미 잘 만드셨습니다.** 온톨로지+KG는 그 위에서 '왜·어떻게·다음에 무엇을'에 답합니다. 챗봇을 대체하는 게 아니라, **의사결정 엔진**을 더하는 겁니다."
