# KOAMI PoC 시연 스크립트 (F011)

> Prototype 리뷰 시연용 **핵심 흐름 outline** (25~35분). 대상: **한국기계산업진흥회(KOAMI) 소부장·가치사슬 담당 + 회원사**. 발표자: 서민원 책임.
> 핵심 목표: "GIVC 위 온톨로지" 가치 + **실 ChatGIVC를 데모로 재현**해 보여주고, **고객이 자발적으로 보유 데이터를 꺼내게** 유도.
> 사전: 라이브 URL 가용 확인(`bash scripts/healthcheck.sh <URL>`) + 오프라인 백업 대기(`pnpm serve:offline`).

---

## 흐름 한눈에

```
① Landing (2~3분) → ② S4 R&D 추천 (8~12분, 메인) → ③ S6 공작기계 가치사슬 + 토글 + ChatGIVC 질의 (10~14분, 신규 하이라이트) → ④ About (3~4분) → ⑤ 마무리·데이터 유도 (3~5분)
```

데이터 표기 강조: 모든 화면에서 **⭐실 / △추정 / ※가상**을 짚어 신뢰성 선점.
서사 한 줄: **"GVC+온톨로지 → 토글로 전 산업 → 실 ChatGIVC 질의 재현 → 본사업화 = Repository 1지점 교체."**

## ① Landing — 데모의 위치 (2~3분)

- **화면**: `/` (Landing)
- **조작**: 히어로 → 시나리오 카드 소개
- **핵심 메시지**:
  - "설문 대신 데모로 — KOAMI GIVC 보유 데이터만으로 가능한 수준 + 추가 데이터 결합 시 확장 path를 한 화면에. 소부장 자립화 의사결정 지원."
  - ProvenanceLegend(⭐△※) 가리키며 "모든 수치에 출처를 답니다. 가상은 ※로 명시 — 신뢰성 오인 없습니다."
- **전환**: "메인 시연 S4부터 보시죠" → S4 카드 클릭

## ② S4 — R&D 적합 기업 추천 (8~12분, 메인)

- **화면**: `/scenario/rnd` (소부장 기계 가치사슬 — 감속기 국산화 등)
- **조작 순서**:
  1. **Top 5 표** 제시 — "산기평 공고가 뜨면 소부장 적합 기업 Top5 + 근거 + 반대 추천이 자동 생성됩니다."
  2. **가중치 슬라이더** 조작(특허 0.30→0.15) — "가중치를 바꾸면 순위가 **실시간 재계산**(<100ms)." (순위 변동 시연)
  3. **선정근거 지식그래프** — 행 hover → 근거 노드 하이라이트, 노드 hover → 출처 tooltip
  4. **Data Expansion Hints** 토글 "산기평 공고 실데이터" ON — "정확도가 **65% → 88%**로. 데이터 한 종을 더 얹었을 때의 효과."
  5. (선택) **What-If 토글** ON → "특허 가중치를 낮추면?" — "실 LLM이 GIVC 컨텍스트로 답합니다. 시범 기능·세션당 3회. (실패 시 정적 응답 자동 대체 — 당황 금지)"
- **데이터 유도 포인트** 🎯:
  - C1: "여기 특허 수는 ※가상 — **특허청 데이터**가 있으면 실값으로." → *특허 데이터 보유 여부 유도*
  - C2: "산기평 공고 텍스트가 있으면 매칭 정확도가 올라갑니다." → *공고 데이터 접근성 유도*

## ③ S6 — 공작기계 가치사슬 + 도메인 토글 + ChatGIVC 질의 (10~14분, 신규 하이라이트)

- **화면**: `/scenario/s6`
- **조작**:
  1. **다단계 지식그래프** — "공작기계(머시닝센터) 중심 전후방 가치사슬: 장비 ← 부품(베어링·감속기) ← 소재(특수강). HS코드·수입국·핵심기업·전후방을 한 화면에 — '창 띄웠다 끄는' 페인을 통합으로."
  2. **수출입 추이 + KPI** — "이상치 ±2σ 시그널 자동 표시. 감속기·특수강 무역수지 적자 = **소부장 자립화 과제**가 데이터로 드러납니다."
  3. **tier 기업표** (소재/부품/장비, hover → 그래프 연동)
  4. ⭐ **도메인 토글 (기계 ↔ 반도체)** — "같은 GIVC 엔진이 **전 산업**에. 공작기계에서 반도체(실리콘웨이퍼)로 토글하면 **동일 분석이 그대로 재렌더**됩니다. 품목 무관 — GVC코드만 바뀌면 됩니다."
  5. ⭐ **💬 ChatGIVC 질의 탭** — "이게 **실 ChatGIVC가 답하는 방식**입니다. 좌 = 실 GIVC SQL(scmm·product_network), 우 = 데모 미러 재현 — 결과는 실제 계산값(⭐). 질문을 골라보세요(매출증가율·전후방·수입의존도). 본사업화 시 **좌측 실 SQL이 귀측 GIVC DB에 직결**됩니다."
- **데이터 유도 포인트** 🎯:
  - C3: "전후방·지표는 공개데이터+가상 재현입니다 — **귀측 GIVC 실데이터**를 연결하면 이 화면이 그대로 실값으로." → *GIVC 연동 유도*
  - C4: "반도체로 토글되는 건 GVC코드 체계가 도메인 무관이라서입니다 — 어떤 품목군을 먼저 보고 싶으신가요?" → *관심 품목 유도*
